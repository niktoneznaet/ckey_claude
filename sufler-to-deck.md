# /sufler-to-deck — суфлёр → слайды

Берёт текст суфлёра (то что произнесёт ведущий) и превращает в:
1. **enriched-суфлёр.md** — тот же текст с inline-маркерами `[SLIDE N: type — content hint]` чтобы ведущий понимал когда меняется кадр
2. **slide-list.json** — структурированный список для `/deck-build`, который рендерит финальные HTML/PDF

## Pipeline где ты живёшь

```
Текст суфлёра (.md) → [ТЫ] → enriched-суфлёр.md + slide-list.json → /deck-build → presentation.html + presentation.pdf
```

## Источник правды

- **Методология подачи:** AlexxxFX-derived block-reset pattern (см. ниже)
- **Brand-frame:** CK Design System v3 (`Бизнес OS/knowledge-base/references/ck-brand/`)
- **Schema совместима с `/deck-build`** — 20 типов слайдов (см. секцию "Type catalog" ниже)

## Что подаётся на вход

- Путь к .md файлу с суфлёром (приоритетно)
- Или текст суфлёра вставленный в чат

Если не указан — спроси у пользователя.

## Что выдаёшь

1. `enriched-сufler.md` — суфлёр + inline `[SLIDE N: ...]` маркеры
2. `slide-list.json` — JSON слайд-листа
3. `examples-brief.md` — бриф под каждый production_note (что найти / снять / сгенерить)
4. `README.md` — короткая инструкция как передать всё в `/deck-build`

**Где сохранять:** `Бизнес OS/outbox/sufler-to-deck/ГГГГ-ММ-ДД_название/`

## Stage 1 — Topic-router (auto-detect)

Прочитай весь суфлёр и определи **тип контента** по сигналам:

### `концепт` — обучающее/методическое видео
Сигналы: автор разбирает абстрактное понятие → структура → выводы. Много "почему", "что это значит", "как это работает". Минимум конкретных продуктов и брендов. Часто 4-6 sub-блоков внутри одной большой темы.
**Slide kit:** centered-intro паттерн доминирует, мало real-charts, нет UI-mockups, comparison-matrix для сопоставления подходов, key-value-table для приземления.

### `метод` — техническая методология / алгоритм
Сигналы: пошаговый разбор техники, конкретные правила входа/выхода, "А → B → C". Часто живая практика / бэктест посередине. Много терминов, иногда формулы.
**Slide kit:** hierarchy-tree для декомпозиции концепта, real-chart placeholder (через `example-slot`), pipeline-flow для алгоритма, outcomes-row для последовательности исходов, прогрессивные 3-кадровые серии (один scaffold + накопление информации), сегмент без слайдов для live-демонстрации.

### `индустрия` — обзор продуктов / рынка / индустрии
Сигналы: имена компаний, статистика, "X% получают Y", "в 2021 было Z, в 2026 W", сравнение брендов, рекомендации по выбору.
**Slide kit:** stats-grid и progress-bar для статистики, screenshot-showcase для UI-доказательств, comparison для брендов, key-value-table для финплана, quote для авторитетов, AI-метафоры через `example-slot` для эмоциональных пиков.

**Если не уверен** — выбери `концепт` (самый универсальный) и сообщи пользователю.

## Stage 2 — Сегментация суфлёра на блоки

Разрежь текст на смысловые блоки. Сигналы границы блока:
- Транзишн-фраза ("Переходим ко второму блоку", "Давайте разберём", "Теперь поговорим")
- Смена темы (новое понятие / новый объект разбора)
- "Итак", "Хорошо", "Так" в начале абзаца
- Перечисление "У нас сегодня четыре блока" → каждый из этих 4 = блок

**Цель:** 4-6 крупных блоков (то что попадёт в TOC).

Если сегментация даёт меньше 3 или больше 7 — пересмотри (слишком крупные или слишком мелкие). Если суфлёр действительно требует 8+ блоков — это уже курс, не одна презентация.

**Для каждого блока зафиксируй:**
- `block_id` (01, 02, …)
- `block_title` (короткое название, 2-4 слова)
- `block_icon_hint` (что хочется на divider — chart/brain/document/pencil/warning/lightbulb/key)
- `start_line` / `end_line` в суфлёре
- `sub_blocks` (если внутри блока есть отдельные под-темы ≥3 — нужны L2-dividers)

## Stage 3 — Классификация сегмента → тип слайда

Для каждого smysловой sub-сегмента внутри блока определи тип слайда по сигналам в речи.

### Триггеры → типы слайдов

| Что говорит автор | Тип слайда (одного из 20 в /deck-build) |
|---|---|
| Открывающая фраза приветствия + тема | `section-title` (cover) |
| "Сегодня у нас N блоков", перечисление пунктов плана | `icon-grid` или `numbered-grid` (N=2-6) для TOC |
| "Переходим ко второму блоку", "Начнём с X" | `section-title` (divider) с иконкой |
| Первые 2-3 предложения после divider'а — тезис блока | `centered-intro` |
| "Я рассматриваю N метрик", "Есть N факторов" с явным числом | `icon-grid` (без номеров если равноправные) или `numbered-grid` (если шаги) |
| "С одной стороны… с другой", "До X / После Y", "Аматоры / Профи" | `comparison` |
| "Алгоритм: A → B → C → D", "Пять шагов" | `steps` (вертикальный список) или `pipeline-flow` (горизонтальная цепочка) |
| Драматическая фраза, повторённая 2 раза с эмоцией | `punch-line` (с lime accent на ключевом слове) |
| Цитата стороннего авторитета ("как говорил Tom Dante…") | `quote` |
| Конкретная цифра/результат как кульминация (+34R, 62%, -7%) | `number` |
| Стат-блок "10-30% / 2-5% / 0.5-2%" | `stats-grid` |
| "Вот сейчас на графике", "посмотрим на кривую" | `example-slot` (kind: chart) |
| "Это пост в Телеграме", "вот письмо от FTMO", "если зайдёте на сайт" | `screenshot-showcase` или `example-slot` (kind: ui-mockup) |
| "Сейчас покажу на TradingView" (длинный кусок ≥5 мин) | НИ ОДНОГО слайда — отметить в production_note "live demo" |
| Определение термина ("FVG — это…", "Narrative — это…") | `term-card` |
| Разложение концепта на 2-3 части ("X состоит из Y и Z") | `hierarchy-tree` |
| Серия исходов ("пять убытков -1000 и два профита +3500…") | `outcomes-row` |
| Конкретный план в виде Проблема / Данные / Решение / KPI | `key-value-table` |
| Метрика-прогресс ("выполнили 7 из 9 пунктов") | `progress-bar` |
| Финальная мысль с продажей ("Workshop стартует тогда-то") | `cta` |
| Эмоциональная метафора без конкретики ("как штанга", "как лестница в горах") | `example-slot` (kind: ai-metaphor) с промптом для image-gen |
| Резюме всех блоков перед заключением | `numbered-grid` recap (повтор TOC с выводами) |
| Финал: главная мысль + одна строка | `takeaway` |

### Важно

- **Никогда** не делай больше одного `punch-line` подряд — будут слипаться визуально.
- **Чарты НИКОГДА не голые** — `example-slot` с описанием должен содержать `hint` что аннотировать.
- **Bulleted-листы запрещены** — конвертируй в `icon-grid` / `numbered-grid` / `comparison`.

## Stage 4 — Применение макроструктуры

После классификации сегментов **достроить структуру** обязательными слайдами:

### Block-reset pattern (повторяется на каждом блоке)

```
1. section-title (divider, иконка + название блока)
2. centered-intro (тезис блока 2-4 строки)
3. контентные слайды (типы из Stage 3)
4. ЗАКРЫТИЕ блока:
   - punch-line ИЛИ
   - key-value-table (конкретные выводы) ИЛИ
   - artefact-слайд (comparison/numbered-grid с итогами)
```

### L1 vs L2 dividers

- **L1 divider** = на каждый из 4-6 главных блоков (один на блок)
- **L2 divider** = если внутри блока ≥3 sub-темы (опционально, добавляется только при необходимости)

Различие только в `caption_size` (L2 чуть меньше), это передаётся через свойство `divider_level: "L1"|"L2"` в JSON.

### Обязательные слайды в начале и в конце deck'а

**В начале:**
1. `section-title` cover (тема + бренд)
2. `icon-grid` / `numbered-grid` TOC (если 4 блока — 2×2, если 5 — 5-tile, если 6 — 3×2)

**В конце:**
1. `numbered-grid` recap (повтор TOC с финальными выводами по каждому блоку)
2. `pipeline-flow` финальный алгоритм (необязательно, если есть смысловой алгоритм)
3. `takeaway` главная финальная мысль (необязательно)
4. `cta` (только если в суфлёре есть призыв на продукт)
5. `section-title` outro (просто wordmark CK, передаётся как `style: "outro"`)

## Stage 5 — Контент-экстракция под каждый тип

Для каждого слайда вытаскиваем из суфлёра конкретный контент.

### Правила извлечения

| Тип | Что извлечь |
|---|---|
| `section-title` | `text` — название блока (2-5 слов), `icon_hint` — что нарисовать |
| `centered-intro` | `segments[]` — массив `{text, style}`, style=muted/primary/accent (lime). 2-4 строки максимум |
| `icon-grid` | `tiles[]` — массив `{glyph, label, sub}`, layout=N колонок |
| `numbered-grid` | `tiles[]` — массив `{num, title, desc}`, layout=N колонок |
| `comparison` | `left: {title, items[]}, right: {title, items[]}` |
| `steps` | `steps[]` — массив `{num, title, desc}` |
| `pipeline-flow` | `nodes[]` — массив строк, `accent_index` (опц), `caption` |
| `punch-line` | `parts[]` — массив `{text, accent: "lime"\|"red"\|"white"\|null}` |
| `quote` | `text`, `attribution` |
| `number` | `value`, `caption` |
| `stats-grid` | `stats[]` — массив `{value, label}` (3-4 элемента) |
| `term-card` | `term`, `expansion`, `note` |
| `hierarchy-tree` | `parent`, `children[]`, `caption` |
| `outcomes-row` | `pills[]` — массив `{kind: "win"\|"loss", value}`, `total`, `caption` |
| `key-value-table` | `rows[]` — массив `{key, value, accent: bool}` |
| `progress-bar` | `fill_percent`, `counter`, `label` |
| `takeaway` | `text` (одна короткая фраза) |
| `cta` | `headline`, `sub`, `buttons[]` |
| `example-slot` | `kind` (chart/diagram/ui-mockup/ai-metaphor/photo), `desc`, `hint` |
| `screenshot-showcase` | `title`, `kind` (TG-post/iMessage/website/email/dashboard), `desc`, `caption` |

### Когда контент не извлекается чисто

Если из суфлёра неочевидно конкретное содержание (например, метафора "как штанга в зале"):
- Заверни в `example-slot` с `kind: "ai-metaphor"`
- В `desc` запиши что слайд должен передать
- В `production_note` запиши готовый промпт для image-gen


**Правила нумерации:**
- `id`: s001, s002, … (3-значный с лидирующим нулём)
- `n`: целое число слайда по порядку
- `block_id`: B1, B2, … (пустой для cover/TOC/outro)
- Слайды без `voice_ref` (например, recap или outro) — `voice_ref: null`

### enriched-сufler.md

Тот же текст суфлёра + inline-маркеры. Маркер ставится **перед** первым словом сегмента к которому относится слайд.

Формат маркера:
```
[SLIDE 03: section-title divider-L1 — "Проблемы анализа" / icon: warning-triangle]
Начнём мы с проблем анализа. Какие они бывают…

[SLIDE 04: centered-intro — "Отсутствие моментального результата"]
Думаю, что многие из вас сталкивались с проблемой…
```

Маркер выделяется в otdельную строку (\n до и после). Текст суфлёра не меняется ни в одной букве — только добавляются маркеры.

### examples-brief.md

Один блок на каждый слайд с `production_note != null`. Формат:

```markdown
## Slide 15 (s015) — example-slot · chart

**Voice ref:** строки 240-255, "Посмотрим на кривую"

**Что нужно:** Кривая капитала за квартал с тремя зонами просадки.

**Подсказка к аннотациям:** Зеленая зона = устойчивый рост, серая = стагнация, красная = просадка.

**Кому передать:** монтажник (скачать chart EURUSD W1 за период Jan-Mar 2026, наложить аннотации зон)

**Готовый промпт для Higgsfield/Open Design:** —
```

### README.md

Короткая инструкция:

```markdown
# Slide deck для "Название"

## Файлы
- `enriched-сufler.md` — для ведущего во время записи
- `slide-list.json` — для `/deck-build`
- `examples-brief.md` — для монтажника / дизайнера
- `source-сufler.md` — оригинал суфлёра без изменений

## Дальше
1. Передать `slide-list.json` в `/deck-build` → получишь `presentation.html` + `presentation.pdf`
2. Передать `examples-brief.md` монтажнику для подбора графиков/скринов/AI-картинок
3. Ведущий читает `enriched-сufler.md`, на маркерах меняется кадр в монтаже
```

## Type catalog для /deck-build (20 типов, для справки)

**Универсальные (используются во всех topic-режимах):**
- `section-title` — divider / cover / outro
- `centered-intro` — параграф-тезис после divider'а
- `icon-grid` — N-tile сетка без номеров
- `numbered-grid` — N-tile сетка с цифрами
- `comparison` — 2-col сравнение
- `steps` — вертикальный список шагов
- `pipeline-flow` — горизонтальная цепочка
- `punch-line` — якорь ключевой фразы с lime/red glow
- `takeaway` — короткая финальная мысль
- `cta` — финальный pitch с кнопками

**Под концепт-видео (преимущественно):**
- `term-card` — определение термина
- `key-value-table` — приземление в цифры
- `quote` — цитата авторитета

**Под метод-видео:**
- `hierarchy-tree` — декомпозиция концепта
- `outcomes-row` — серия исходов
- `example-slot` (kind=chart) — placeholder для real-chart'а
- `number` — кульминационная цифра

**Под индустрию:**
- `stats-grid` — 3-4 опорных статистики
- `screenshot-showcase` — UI-доказательство
- `progress-bar` — визуальная метрика
- `example-slot` (kind=ai-metaphor) — эмоциональный пик

## Ритм и плотность (по topic-type)

После сборки проверь итоговое количество слайдов:

| Topic | Среднее сл/мин | Допустимое распределение |
|---|---|---|
| `концепт` | 1.0-1.3 | Равномерно по всем блокам, можно 70% слайдов на одном главном блоке если в нём вся ценность |
| `метод` | 3-5 в теории, 0 в live-демо, 1-2 в заключении | Резкие зоны пустоты — это норма |
| `индустрия` | 1.5-2.0 | Плотность падает к концу. Психологический блок — обычно пик |

Если на 10-минутный суфлёр выходит ≥80 слайдов — слишком плотно, упрощай.
Если на 40-минутный суфлёр выходит ≤25 слайдов — слишком пусто, добавь centered-intro / punch-line на пропущенные мысли.

---

## Anti-patterns

❌ **Маркированные списки в слайдах** — конвертируй в icon-grid / comparison / steps
❌ **Длинные тексты (>4 строки) кроме centered-intro** — раздели на 2 слайда или сожми
❌ **Два punch-line подряд** — слипаются визуально, поставь content между ними
❌ **Голые chart-слайды без аннотаций** — всегда указывай `hint` что подсвечивать
❌ **Маркер `[SLIDE]` в середине предложения** — только перед первым словом смыслового сегмента
❌ **Дублирование TOC и recap дословно** — recap должен содержать выводы по каждому блоку, не просто названия
❌ **CTA если в суфлёре нет призыва к продукту** — не вставляй ради ритуала
❌ **`number`-слайд с абстрактной цифрой** — должна быть конкретная метрика с подписью

## Процесс работы

1. **Спроси путь** к суфлёру (если не указан): `.md` файл или вставленный текст.
2. **Прочитай весь суфлёр целиком** — не построчно, а как цельный документ.
3. **Stage 1 — определи topic-type.** Сообщи пользователю и спроси подтверждение если не уверен (концепт/метод/индустрия).
