# Продающая презентация для эфиров

## Задача

Создай или доработай продающий блок презентации (слайды), который используется в конце вебинаров/эфиров для перехода к анкете FTT.

## Входные данные от пользователя

Запроси (если не указано):

1. **Контекст:** после какой подводки идёт этот блок (тема эфира, что уже было)
2. **Продукт:** FTT / другой (описание, цена, бонусы, формат)
3. **Скриншоты текущей презентации** (если есть, для аудита)
4. **Кейсы учеников** (имена, результаты, цифры)
5. **Бонусы и офферы** (предоплата, рассрочка, подарки за быструю запись)

## Структура продающего блока (проверенная)

### Оптимальный порядок слайдов:

1. **Переход от контента к продаже** — мост через самодиагностику (3 вопроса: «если вы ответили да — вам сюда»)
2. **Кто мы** — авторитет команды (1 слайд, коротко)
3. **Три уровня / три месяца** — что ученик умеет ПОСЛЕ каждого этапа (не что внутри, а результат)
4. **Типичная неделя на курсе** — формат: созвоны, ревью, лайв-торговля, бот, Q&A
5. **Снятие возражений** — «А если нет опыта?», «А если не получится?», «Сколько времени в день?»
6. **Результаты учеников** — кейсы с фото, цифрами, историями до/после
7. **Оффер** — ОДИН слайд: что получаешь + сколько стоит + бонусы + дедлайн
8. **CTA** — конкретный следующий шаг: «заполни анкету → мы перезвоним → подберём уровень»

### Чего НЕ делать:

- Не показывать 36 модулей с терминами (Order Flow, Canddle Range, PO3/AMD) — пугает новичка
- Не размазывать оффер на 3 слайда — собрать в один
- Не заканчивать на QR-кодах соцсетей — заканчивать на CTA
- Не ставить «Результат зависит только от тебя» как давление — переформулировать как снятие страха
- Продающий блок на эфире — максимум 5–7 минут. Детали рассказывает отдел заботы на звонке.

## Формат выхода

Для каждого слайда:

```
### СЛАЙД N: [Название]
**Визуал:** Что должно быть на слайде
**Текст:** Заголовок + тезисы
**Что говорит спикер:** Ключевые фразы (2–3 предложения)
**Хронометраж:** X минут
```

Плюс общая памятка для спикера: тайминг, на что обращать внимание, как реагировать на вопросы из чата.
